  • Creer et exploiter
  • Le choix des exercices se fait par niveaux, puis par domaines présentés en onglets. En cliquant sur les différents domaines de chaque niveaux, les exercices apparaitront. L'onglets "Fermer" permet de replier les onglets du niveau correspondant et ainsi d'alléger l'affichage. Chaque ligne d'un onglet correspond à un exercice, on va donc en détailler les différents champs de la gauche vers la droite :
  • La vignette permet de se faire une idée rapide de l'exercice.
  • Le champs de saisie du nombre de question pour cet exercice (au maximum 20).
  • Le libellé de l'exercice.
  • Le(s) champ(s) de saisie de(s) paramètre(s). Ils correspondent en général à la taille des nombres utilisés dans l'exercice et permettent donc d'en faire varier la difficulté.
  • Pour des raisons techniques, vous ne pouvez pas générer des documents contenant plus de 20 questions.
  • La plupart de ces informations sont explicites et figureront sur la page de garde. Malheureusement, pour des raisons techniques, ces informations ne seront pas conservées si vous rechargez la page. Voici un exemple de page de garde en conservant les informations par defaut:
  • L'information la plus importante est le "Temps par slide" qui est le temps en secondes d'affichage de chacune des questions. Veuillez donc bien à l'ajuster au mieux pour ne pas avoir une activité mentale trop rapide ou trop lente.
  • Les sorties dites "Vidéoprojetable" sont destinées à être utilisées en classe avec une vidéo-projecteur. Leurs caractéristiques sont les suivantes:
  • Une question ou un corrigé par slide.
  • Les slides du sujet défilent automatiquement selon le paramètre "Temps par slide" (sauf page de garde)
  • Les différents modèles proposés sont optimisés pour une grande lisibilité.
  • Les sorties dites "Papier" sont destinées à être imprimées. Leurs caractéristiques sont les suivantes:
  • Toutes les questions ou les corrigés sont sur la même page.
  • Les différents modèles proposent une gestion optimale de l'espace (gain de copie).
  • Le choix du modèle est très important. Leurs caractéristiques sont les suivantes:
  • Les modèles monocolonnes proposent une seule question par slide ou page.
  • Les modèles bicolonnes identiques proposent 2 questions par slide ou page. Ces 2 questions disposées côte à côte verticalement sont de même type, mais les valeurs utilisés sont différentes. Cela permet de faire travailler les 2 élèves d'une table sans risque de triche. Attention cependant la difficulté des séries droite et gauche peut être inégale.
  • Les modèles bicolonnes inversées proposent 2 questions par slide ou page. Actimaths Web génère les questions demandées, puis dispose ces questions dans le bon sens à gauche et dans le sens inverse à droite. Cela permet de faire travailler les 2 élèves d'une table en minimisant la triche. Les questions étant les mêmes (mais décallées), il n'y a pas de soucis d'équité.
  • Quand vous avez fait votre choix d'exercices, que vous avez remplis vos informations (optionnel) et sélectionner au moins une sortie, voici le choix qui se présente à vous:
  • Le bouton "Annuler" réinitialise la page.
  • Le bouton "Créer" lance la crétion des documents.
  • Le bouton "Aide" lance la page d'aide que vous êtes en train de lire en ce moment.
  • Le bouton "Crédits" affichent les remerciments et les liens vers les ressources utilisées.
  • Une fois le bouton "Créer" cliqué, Actimaths Web se met au travail. Après un temps de calcul plus ou moins important selon vos choix d'exercices et de sorties, vous devriez avoir une notification de téléchargement pour un fichier nommé "exercice.zip". Par exemple pour Firefox:

Si vous ne parvenez pas à ouvrir ce fichier, vous devez installer un logiciel de compression/décompression (7zip, winrar, winzip...etc). Si tout se déroule bien, vous devriez obtenir une fenètre similaire à celle-ci :
  • Vous pouvez remarquer que Actimaths Web vous renvoie 2 fichiers par sortie sélectionnée. Ainsi, dans notre exemple, j'ai sélectionné les sorties "Sujet vidéoprojetable" et "Corrigé vidéoprojetable". J'obtiens alors 4 fichiers en retour. On va maintenant se concentrer sur les fichiers "exercice-sujet-presentation.tex" et "exercice-sujet-presentation.pdf" qui sont le retour de la sortie "Sujet vidéoprojetable":
  • Le fichier "exercice-sujet-presentation.pdf" est directement lisible par un lecteur de PDF. C'est probablement celui que vous voulez.
  • Le fichier "exercice-sujet-presentation.tex" est le code source laTeX. Si vous ne savez pas à quoi il correspond, ne vous embêtez pas avec lui.
  • On va donc sélectionner le fichier "exercice-sujet-presentation.pdf" pour commencer l'activité mentale ou alors le fichier "exercice-corrige-presentation.pdf" pour la correction. Si vous ne parvenez pas à ouvrir ces fichiers, vous devez installer un logiciel de lecture de PDF (Adode Reader, Foxit...etc). Si tout se déroule bien, vous devriez obtenir une fenètre similaire à celle-ci :
  • Voici quelques précisions pour bien emploiter ces PDF:
  • Les transitions automatiques (souvenez vous le paramètre "Temps par slide") ne seront actives qu'en plein écran. Celui-ci s'active en général avec la touche "F5".
  • La page de garde du sujet n'a pas de transition automatique. Vous pouvez alors attendre que toute la classe se prépare et prenne bien en compte le temps par slide.
  • La correction n'a pas du tout de transition automatique. Vous pouvez prendre le temps que vous voulez pour faire la correction.
  • La sortie "Corrigé papier" peut être projetée dans certain cas pour faire une correction plus rapide.

  • Languages de programmation utilisés
  • Pour développer votre propre exercice, vous devrez maitriser au mieux les 2 languages suivants :
  • Python 2 : c'est le language dans lequel est programmé Actimaths. Il est très puissant du fait de ses nombreux modules tout en restant simple et facile à débugger.
  • LaTeX : Actimaths n'est en fait qu'un générateur automatisé de code LaTeX. Vous devez donc être capable d'obtenir ce que vous voulez avec LaTeX avant d'en automatiser sa production.
  • Dépendances logicielles
  • Avant de vous lancer, vous devez installer certains logiciels indispensables au bon développement d'Actimaths. On peut les décomposer en 3 grands groupes :
  • Un environnement Python 2 : Python 2 lui-même et lxml (un module qui gère les fichiers au format xml).
  • Une distribution LaTeX : Un ensemble qui permet compiler le code LaTeX en PDF.
  • Des éditeurs de code : Un spécialisé pour le code Python et un pour le code LaTeX.
  • Windows :
  • Vous trouverez Python 2 ici et le module lxml ici. Faites bien attention de télécharger la dernière version sortie de Python 2 (actuellement la 2.7.6) et la version de lxml correspondante.
  • Pour ce qui de LaTeX, vous avez le choix entre MiKTeX ou TeX Live. Personnellement, je n'ai testé que MiKTeX.
  • L'éditeur de code Python IDLE qui est installé en même temps que Python 2 est suffisant. Je vous conseille Texmaker pour le code LaTeX. La page de téléchargement de Texmaker contient aussi des explications pour configurer MiKTeX et TeX Live.
  • Mac Os X (à confirmer vu que je n'ai pas de Mac) :
  • Vous trouverez Python 2 ici. Le module lxml s'installe avec la ligne de commande suivanate :
  • STATIC_DEPS=true sudo pip install lxml
  • Pour ce qui de LaTeX, vous devez installer MacTeX .
  • L'éditeur de code Python IDLE qui est installé en même temps que Python 2 est suffisant. Je vous conseille Texmaker pour le code LaTeX. La page de téléchargement de Texmaker contient aussi des explications pour configurer MacTeX.
  • Linux :
  • Sur Ubuntu, vous devez exécuter la commande suivante pour installer les paquets nécessaires :
  • sudo apt-get install python-lxml texlive-pstricks texlive-latex-base texlive-latex-extra texlive-fonts-recommended texlive-latex-recommended
  • Sur Archlinux, vous devez exécuter la commande suivante en root pour installer les paquets nécessaires :
  • pacman -S python2 python2-lxml texlive-pstricks texlive-latexextra

  • Un exercice de pourcentage en sixième
  • Code Python:
  • #!/usr/bin/python
# -*- coding: utf-8 -*-

import random

def parametre():
    objet = ['allumettes','billes','boules','timbres','pieces','crayons']
    i = random.randrange(0, len(objet))
    pourcentage = [10, 20, 25, 50]
    j = random.randrange(0, len(pourcentage))
    part = random.randrange (2, 10)
    total = part * 100 / pourcentage[j]
    return objet[i], part, total, pourcentage[j]

def tex_pourcentage(exo,cor):
    (objet, part, total, pourcentage)= parametre()
    exo.append("Je prends $ %s $ %s parmi $ %s $. \\newline" % (part, objet, total))
    cor.append("Je prends $ %s $ %s parmi $ %s $. \\newline" % (part, objet, total))
    exo.append(u"J\'ai donc pris $ \\ldots \\%% $ des %s" % objet)
    cor.append(u"J\'ai donc pris $ \\boxed{%s \\%%} $ des %s" % (pourcentage, objet))

def CalculPourcentage(parametre):
    question = u"Compléter :"
    exo = []
    cor = []
    tex_pourcentage(exo,cor)
    return (exo, cor,question)

  • Un exercice de lecture d'angle sur le rapporteur en sixième
  • Code Python :
  • #!/usr/bin/python
# -*- coding: utf-8 -*-

import random

#------------------methode---------------------------------------------
def tex_rapporteur(enonce):
    enonce.append("\\psarc(0,0){5}{0}{180}")
    enonce.append("\\psarc(0,0){4.5}{0}{180}")
    enonce.append("\\psarc(0,0){3}{0}{180}")
    enonce.append("\\psarc(0,0){0.5}{0}{180}")
    enonce.append("\\SpecialCoor")
    enonce.append("\\multido{\\i=0+1}{181}{\\psline[linewidth=0.01](4.5;\\i)(5;\\i)}")
    enonce.append("\\multido{\\i=0+10}{19}{\\psline(4;\\i)(5;\\i)}")
    enonce.append("\\psframe(-5,-1)(5,0)")
    enonce.append("\\multido{\\i=5+10}{18}{\\psline(4.25;\\i)(5;\\i)}")
    enonce.append("\\tiny")
    enonce.append("\\multido{\\i=0+10}{19}{\\rput(3.75;\\i){\\i }}")
    enonce.append("\\pscircle[fillstyle=solid, fillcolor=white]{0.1} ")
    enonce.append("\\NormalCoor")


def tex_angle(enonce, angle, nom_angle):
     enonce.append("\\SpecialCoor")
     enonce.append("\\psline[linewidth=0.05](0;0)(5.2;%s)" % angle)
     enonce.append("\\rput(5.5;%s){$%s$}" % (angle, nom_angle))
     enonce.append("\\NormalCoor")

def tex_mesure(enonce, angle, nom_angle):
     enonce.append("\\begin{center}")
     enonce.append("\\psset{unit=0.5cm}")
     enonce.append("\\begin{pspicture}(-5,-1)(5,5.5)")
     tex_rapporteur(enonce)
     for i in range(len(angle)):
         tex_angle(enonce,angle[i],nom_angle[i])
     enonce.append("\\end{pspicture}")
     enonce.append("\\end{center}")

#------------------construction-----------------------------------------
def MesureSimple(parametre):
    choix = random.randrange(2)
    if choix:
        nom_angle = "\\widehat{xOy}"
    else:
        nom_angle = "\\widehat{yOz}"
    question = "Donner la mesure de l'angle $%s$ :" % nom_angle 
    exo = []
    cor = []
    angle = random.randrange(5,175,5)
    tex_mesure(exo, [0,angle,180],["x","y","z"])
    tex_mesure(cor, [0,angle,180],["x","y","z"])
    if choix:
        cor.append("$$ %s= %s^\\circ$$ " %(nom_angle, angle))
        cor.append("La mesure de $%s$ est $%s^\\circ$ " %(nom_angle, angle))
    else:
        cor.append("$$ %s= %s^\\circ - %s^\\circ$$ " %(nom_angle, 180, angle))
        cor.append("$$ %s= %s^\\circ$$ " %(nom_angle, 180 - angle))
        cor.append("La mesure de $%s$ est $%s^\\circ$ " %(nom_angle, 180 - angle))          
    return (exo, cor, question)
